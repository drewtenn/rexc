# The Rexy Programming Language

*A Tutorial*

Rexy is the language side of the Drunix experiment. Drunix asks what it
takes to build an operating system from the first boot instruction
upward. Rexy asks the companion question: what does a small, modern
systems language for that operating system look like, and how do you
learn to write in it?

This book is for programmers who already know at least one language. It
teaches Rexy by example. Every chapter gives you code to type, a way to
run it, and the output you should see. By the end of the book you will
have written, run, and modified Rexy programs that touch every feature
the language currently provides.

You will see Rexy as a systems language. Memory is visible. Mutation is
opt-in. Pointer access lives behind an `unsafe` boundary. Types are
written down. The book covers those choices on purpose, because they
are the choices that make Rexy a useful language for an operating
system to host.

**Drew Tennenbaum**
